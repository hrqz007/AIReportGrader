# 内置文档预览转换器说明

实验智评 V2 的教师复核页会优先把 `.docx` 报告转换为 PDF 后预览，这样比直接解析 Word XML 生成 HTML 更接近原始报告版式。

## 转换器查找顺序

后端会按以下顺序查找转换器：

1. 环境变量 `AIGRADER_OFFICE_CONVERTER` 指向的 `soffice.exe`；
2. 已解压的用户运行缓存 `%LOCALAPPDATA%/AIReportGraderV2/libreoffice/program/soffice.exe`；
3. 项目内置目录 `runtime/libreoffice/program/soffice.exe`；
4. 项目内置目录 `runtime/LibreOffice/program/soffice.exe`；
5. 项目内置目录 `runtime/LibreOfficePortable/App/libreoffice/program/soffice.exe`；
6. 项目内置目录 `tools/libreoffice/program/soffice.exe`；
7. 项目内置压缩包 `runtime/libreoffice.zip`，首次预览时自动解压到用户运行缓存目录；
8. 系统 PATH 中的 `soffice` 或 `libreoffice`；
9. Windows 常见安装目录下的 LibreOffice；
10. Windows Word COM 自动化能力作为兜底。

## 推荐打包方式

为了让其他老师不必安装 Word、WPS 或 LibreOffice，建议在发布压缩包前将 LibreOffice 安装目录复制到：

```text
runtime/libreoffice/
```

复制后应存在：

```text
runtime/libreoffice/program/soffice.exe
```

当前项目的 `scripts/package-release.ps1` 会复制整个 `runtime/` 目录，因此该转换器会随便携版一起发布。

为了避免 Python 语法检查扫描 LibreOffice 自带的 Python 运行库，也可以将该目录压缩为：

```text
runtime/libreoffice.zip
```

系统检测到该压缩包后，会在首次生成 Word 预览时自动解压到 `%LOCALAPPDATA%/AIReportGraderV2/libreoffice/`。这样既能随系统分发，又不会污染项目源码目录和 Python 语法检查。

## 预览降级策略

如果没有检测到可用转换器，系统不会中断教师复核流程，而是自动降级为 HTML 预览，并在页面中提示版式可能不准确。此时仍可下载原始 Word 报告核对。

## 注意事项

- 不建议依赖 WPS 自动转换，因为不同版本和安装方式的可调用能力不稳定。
- 内置 LibreOffice 会增加压缩包体积，但能显著提升“解压即用”的可靠性。
- PDF 预览结果会缓存在 `backend/exports/previews/`，同一报告不会每次重复转换。
